# Documentación del Proyecto

Esta es la carpeta más importante para entender el proyecto.

## Archivos clave (leelos en este orden)

| Orden | Archivo | Qué vas a encontrar |
|-------|---------|---------------------|
| 1 | `REGLAS_DEL_SISTEMA.md` | Todas las reglas de negocio, roles, flujos, permisos y cómo debe comportarse el sistema. |
| 2 | `PROJECT_STRUCTURE.md` | Explicación clara y amigable de cómo está organizado el código. Ideal para navegar en VS Code. |
| 3 | `PLAN_DE_DESARROLLO.md` | En qué orden vamos construyendo el sistema (fases). |

## Recomendación

Cuando abras el proyecto en VS Code y no entiendas algo, vení primero a esta carpeta. Es mucho más fácil leer la explicación en español que tratar de descifrar el código directamente.

**Nota:** Esta documentación se mantiene actualizada a medida que el proyecto evoluciona.
